📋 Design Notes for Quickfox Team

Product Identification & Search

  • • Product ID: Unique identifier (e.g., PRD-00142) — auto-generated or hospital-assigned
  • • Search indexes: product_id, generic_name, brand_name, atc5 — all searchable
  • • Autocomplete: Show ID, generic, brand, manufacturer, route, pack size in results
  • • Highlight matching: Yellow highlight on matched text in search results
  • • Pre-filled fields: When product selected, auto-fill strength, packsize, route from catalog
  • • User only enters: Quantity (packs) — everything else comes from product master

Data Entry Principles

  • • Instant DDD calculation: Show computed DDDs in real-time as quantity changes
  • • AWaRe badges: Color-code entries immediately (green/amber/red)
  • • Draft saving: Auto-save every 30 seconds
  • • Session management: Group entries by ward + period
  • • Add new product: Link to add product to catalog if not found

Validation Flow

  • • Data Entry Specialist: Creates entries (status: pending)
  • • Data Manager: Reviews and validates or rejects
  • • Admin: Can edit validated data, delete permanently
  • • Plausibility checks: Flag unusual quantities, missing data

File Upload Supported Formats

  • • Excel (.xlsx, .xls): WHO template format, custom hospital formats
  • • CSV: UTF-8, comma or semicolon delimited
  • • Pharmacy System Exports: Custom parsers for common systems
