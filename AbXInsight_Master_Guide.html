üß™ Experimental Features (AI Data Extraction)

The AI Data Extraction feature uses AI to crawl through any uploaded dataset and attempt to extract relevant antimicrobial consumption data. This is marked as EXPERIMENTAL because:

  • ‚Ä¢ AI extraction may contain errors and requires human verification
  • ‚Ä¢ Works best with structured data but can attempt unstructured formats
  • ‚Ä¢ Users must review and confirm all extracted data before saving
  • ‚Ä¢ Extraction confidence scores should be displayed for each field

Quickfox Implementation:

  • ‚Ä¢ Add "AI Extract" button next to file upload (PRO+ only)
  • ‚Ä¢ Show extraction preview with confidence indicators (green/amber/red)
  • ‚Ä¢ Allow users to edit/correct extracted values before confirming
  • ‚Ä¢ Display warning banner: "AI-extracted data - please verify before saving"

6. Implementation Checklist

Step-by-step development tasks

Phase 1: Core Infrastructure

  • Set up routing with /hospital/ and /country/ prefixes
  • Implement authentication (login/register)
  • Build tier-based access control middleware
  • Create user database with module & tier fields
  • Build Dashboard Selection page (/select-dashboard)

Phase 2: Hospital Module

  • Build Hospital Home (/hospital/home)
  • Build Hospital Dashboard with tier locking
  • Build Data Entry screens (all 5)
  • Build Antibiotic Catalogue (all 3 screens)
  • Build Reports section (Pro+ only)
  • Build AI Agent section (Ultra only)
  • Build Settings screens

Phase 3: Country Module

  • Build Country Home (/country/home)
  • Build Country Dashboard with tier locking
  • Build Facility Registry
  • Build WHO GLASS Export
  • Build Country Reports (Pro+ only)

  • Implement Stripe payment integration
  • Build upgrade/downgrade flow
